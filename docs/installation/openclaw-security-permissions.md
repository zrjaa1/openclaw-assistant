# OpenClaw 安全权限配置指南

安装和初始配置完成后，强烈建议你在正式使用之前，先给小龙虾（OpenClaw）设置一个合适的权限级别。这一步非常重要 —— 它决定了小龙虾能在你的电脑上做哪些事情。

> ⚠️ 重要提示：小龙虾默认使用「full」（完全权限）模式，这意味着它可以不受限制地执行任何操作。建议你在开始使用之前，先按照本指南设置更安全的权限。

---

## 什么是权限级别？

小龙虾（OpenClaw）通过「工具配置文件」（Tool Profile）来控制 AI 助手能使用哪些功能。你可以把它想象成手机上的"应用权限" —— 你可以选择让它访问哪些功能，不让它访问哪些功能。

一共有 4 个权限级别，从最严格到最宽松：

---

### 1. minimal（最小权限）

AI 助手只能查看自己的运行状态，不能做任何其他事情。

适合谁：只想测试小龙虾能不能跑起来，暂时不想让它做任何实际操作的用户。

能做的事：
- 查看运行状态

不能做的事：
- 不能读写文件
- 不能执行命令
- 不能发送消息
- 不能上网搜索
- 基本上什么都不能做

---

### 2. messaging（消息权限）— Windows 用户推荐

AI 助手可以收发消息、查看对话历史，但不能操作你电脑上的文件或执行命令。

适合谁：把小龙虾当作聊天助手使用的 Windows 用户。只需要它在微信、飞书、钉钉里回复消息，不需要它操作电脑上的文件。

能做的事：
- 收发消息（微信、飞书、钉钉等）
- 查看对话历史
- 查看运行状态

不能做的事：
- 不能读写你电脑上的文件
- 不能执行终端命令
- 不能上网搜索
- 不能控制浏览器

---

### 3. coding（编程权限）— Mac/Linux 用户推荐

AI 助手可以读写文件、执行命令、管理任务，是开发者和进阶用户的理想选择。

适合谁：希望小龙虾帮你处理文件、写代码、执行脚本的 Mac 或 Linux 用户。

能做的事：
- 读写文件
- 执行终端命令
- 管理对话和子任务
- 使用记忆功能（记住你告诉它的事情）
- 分析图片

不能做的事：
- 不能直接发送消息到你的聊天工具
- 不能控制浏览器
- 不能管理定时任务

---

### 4. full（完全权限）— 需要谨慎使用

AI 助手可以使用所有功能，没有任何限制。这是小龙虾的默认设置。

适合谁：完全信任小龙虾，并且清楚了解所有功能含义的高级用户。

⚠️ 注意：「full」模式意味着小龙虾可以不受任何限制地操作你的电脑，包括：
- 读写任何文件
- 执行任何命令
- 发送消息
- 控制浏览器
- 上网搜索
- 管理定时任务
- 控制连接的设备

除非你完全了解这些权限的含义，否则不建议使用此模式。如果你不确定，请先使用上面推荐的 messaging 或 coding 模式。

---

## 如何设置权限？

### 方法一：编辑配置文件（推荐）

1. 找到你的小龙虾配置文件。配置文件的位置是：

Windows:
%USERPROFILE%\.openclaw\openclaw.json

Mac/Linux:
~/.openclaw/openclaw.json

2. 用文本编辑器（Windows 可以用记事本，Mac 可以用 TextEdit）打开这个文件

3. 找到或添加下面的内容：

Windows 用户推荐设置（messaging 模式）：

{
  "tools": {
    "profile": "messaging"
  }
}

Mac/Linux 用户推荐设置（coding 模式）：

{
  "tools": {
    "profile": "coding"
  }
}

4. 保存文件

5. 重启小龙虾让设置生效：

openclaw gateway restart

### 方法二：用命令设置

如果你觉得编辑文件太麻烦，也可以直接运行命令：

openclaw config set tools.profile messaging

或者：

openclaw config set tools.profile coding

然后重启：

openclaw gateway restart

## 进阶：命令执行审批

除了权限级别之外，你还可以设置「命令执行审批」，让小龙虾每次要执行命令时都先问你一声。这就像手机 App 每次要用摄像头时都会弹窗请求许可一样。

在配置文件中添加：

{
  "tools": {
    "profile": "coding"
  },
  "approvals": {
    "exec": {
      "enabled": true
    }
  }
}

这样每次小龙虾想执行命令时，都会先在聊天中问你是否允许。你可以选择：
- 允许这一次
- 永远允许这个命令
- 拒绝

## 权限级别对比速查

minimal — 只能看状态，什么都不能做
messaging — 能聊天，不能动文件（Windows 推荐）
coding — 能读写文件和执行命令（Mac/Linux 推荐）
full — 什么都能做，没有限制（默认，谨慎使用）

## 常见问题

Q：设置了权限后，以后想改怎么办？
A：随时可以修改配置文件或重新运行 openclaw config set 命令来更改权限级别。改完后记得运行 openclaw gateway restart。

Q：我选了 messaging 但后来想让它帮我处理文件怎么办？
A：把权限改成 coding 就行，参考上面的设置方法。

Q：为什么默认是 full？不是很不安全吗？
A：默认 full 是为了让高级用户开箱即用。但对于大多数用户来说，messaging 或 coding 已经足够，而且更安全。所以我们建议你在开始使用前先设置一个合适的权限。
